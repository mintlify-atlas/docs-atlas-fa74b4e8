---
title: "Configuring Rails Applications"
description: "Learn how to configure Rails applications using configuration files, initializers, and the Rails configuration API."
---

Rails offers four standard spots to place initialization code:

- `config/application.rb` — application-wide defaults
- Environment-specific configuration files (`config/environments/*.rb`)
- Initializers (`config/initializers/*.rb`)
- After-initializers (`config.after_initialize`)

## Configuration Files Overview

<Accordion title="config/application.rb">
The primary configuration file for your application. It requires the framework, loads gems, and sets defaults that apply across all environments.

```ruby
require_relative "boot"

require "rails/all"

Bundler.require(*Rails.groups)

module MyApp
  class Application < Rails::Application
    # Load defaults for the current Rails version
    config.load_defaults 8.2

    # Add lib/ to autoload paths (ignore non-Ruby subdirectories)
    config.autoload_lib(ignore: %w[assets tasks])

    # config.time_zone = "Central Time (US & Canada)"
    # config.eager_load_paths << Rails.root.join("extras")
  end
end
```

Settings in this file are inherited by all environments. Environment-specific files processed later can override them.
</Accordion>

<Accordion title="config/environments/*.rb">
Each environment has its own file that overrides application-wide defaults. Rails ships with three: `development.rb`, `test.rb`, and `production.rb`.

```ruby
# config/environments/production.rb
Rails.application.configure do
  config.enable_reloading = false
  config.eager_load = true
  config.consider_all_requests_local = false
end
```
</Accordion>

<Accordion title="config/initializers/">
Files in `config/initializers/` are loaded after the framework and gems are initialized. Use them for configuration that must run after all frameworks are ready.

```ruby
# config/initializers/filter_parameter_logging.rb
Rails.application.config.filter_parameters += [
  :passw, :email, :secret, :token, :_key, :crypt, :salt,
  :certificate, :otp, :ssn, :cvv, :cvc
]
```

<Note>
There is no guarantee your initializers run after all gem initializers. If your code depends on a specific gem being ready, use `config.after_initialize` instead.
</Note>
</Accordion>

<Accordion title="config.after_initialize">
Takes a block that runs after Rails finishes initializing the application—including the framework, engines, and all initializers. Also runs for Rake tasks.

```ruby
config.after_initialize do
  ActionView::Base.sanitized_allowed_tags.delete "div"
end
```
</Accordion>

## Versioned Default Values

`config.load_defaults` loads default configuration values for a target version and all prior versions. For example, `config.load_defaults 8.2` applies defaults from all versions up through 8.2.

```ruby
config.load_defaults 8.2
```

<Warning>
Use the public configuration methods over calling directly to the associated class. Use `Rails.application.config.action_mailer.options` instead of `ActionMailer::Base.options`.
</Warning>

## Rails General Configuration

The following options are called on the `Rails::Application` config object.

<Accordion title="Core Application Settings">
| Option | Default | Description |
|---|---|---|
| `config.time_zone` | `"UTC"` | Sets the default time zone and enables time zone awareness for Active Record |
| `config.encoding` | `Encoding::UTF_8` | Application-wide encoding |
| `config.beginning_of_week` | `:monday` | Default start of week for date calculations |
| `config.log_level` | `:debug` (`:info` in production) | Logger verbosity; options are `:debug`, `:info`, `:warn`, `:error`, `:fatal`, `:unknown` |
| `config.log_file_size` | `104_857_600` (100 MiB) in dev/test | Maximum Rails log file size in bytes; unlimited in production |
| `config.filter_parameters` | `[]` | Parameters to filter from logs and `#inspect` output |
| `config.eager_load` | `nil` | When `true`, eager loads all registered `config.eager_load_namespaces` |
| `config.enable_reloading` | `true` in development | Reloads classes/modules between requests when they change |
| `config.api_only` | `false` | Loads a smaller middleware stack suitable for API-only apps |

```ruby
# Set application time zone
config.time_zone = "Central Time (US & Canada)"

# Set log level from environment variable
config.log_level = ENV.fetch("RAILS_LOG_LEVEL", "info")

# Filter sensitive parameters from logs
config.filter_parameters += [:password, :credit_card_number]
```
</Accordion>

<Accordion title="SSL and Security Settings">
| Option | Default | Description |
|---|---|---|
| `config.force_ssl` | `false` | Forces all requests over HTTPS via `ActionDispatch::SSL` middleware |
| `config.assume_ssl` | `false` | Makes the app treat all requests as arriving over SSL; useful behind SSL-terminating load balancers |
| `config.require_master_key` | `false` | Prevents boot if `ENV["RAILS_MASTER_KEY"]` or `config/master.key` is missing |
| `config.hosts` | `[]` (all in development) | Array of strings, regexps, or `IPAddr` values used to validate the `Host` header |

```ruby
# Allow requests from a domain and its subdomains
config.hosts << ".example.com"
config.hosts << /.*\.product\.com/

# Force SSL in production
config.force_ssl = true
```
</Accordion>

<Accordion title="Caching and Sessions">
| Option | Default | Description |
|---|---|---|
| `config.cache_store` | `[:file_store, "#{root}/tmp/cache/"]` | Cache store for Rails caching; options include `:memory_store`, `:file_store`, `:mem_cache_store`, `:null_store`, `:redis_cache_store` |
| `config.session_store` | cookie store with app name key | Class used to store sessions |
| `config.public_file_server.enabled` | `true` | Whether Rails serves static files from `public/` |

```ruby
# Use memory store for caching
config.cache_store = :memory_store

# Use cookie store with a custom key
config.session_store :cookie_store, key: "_myapp_session"

# Use Redis for caching
# config.cache_store = :mem_cache_store
```
</Accordion>

<Accordion title="Autoloading and Eager Loading">
| Option | Default | Description |
|---|---|---|
| `config.autoload_paths` | `[]` | Paths Rails autoloads constants from; not recommended to adjust since Rails 6 |
| `config.autoload_once_paths` | `[]` | Paths autoloaded but not reloaded between requests |
| `config.eager_load_paths` | All `app/` subdirectories | Paths eager loaded when `config.eager_load` is `true` |
| `config.add_autoload_paths_to_load_path` | `false` (since 7.1) | Whether autoload paths are added to `$LOAD_PATH` |

```ruby
# Add lib/ to autoload and eager load paths
# (pass any non-Ruby subdirectories to ignore:)
config.autoload_lib(ignore: %w[assets tasks generators])

# Add lib/ to autoload-once paths (not reloaded)
config.autoload_lib_once(ignore: %w[assets tasks])
```
</Accordion>

<Accordion title="Logging">
`config.logger` defaults to `ActiveSupport::TaggedLogging` wrapping `ActiveSupport::Logger` writing to `log/<env>.log`.

```ruby
# Custom logger
class MyLogger < ::Logger
  include ActiveSupport::LoggerSilence
end

mylogger           = MyLogger.new(STDOUT)
mylogger.formatter = config.log_formatter
config.logger      = ActiveSupport::TaggedLogging.new(mylogger)

# Tag log lines with request ID
config.log_tags = [ :request_id ]

# Silence health check logs
config.silence_healthcheck_path = "/up"
```
</Accordion>

<Accordion title="YJIT">
As of Rails 7.2 with Ruby 3.3+, YJIT is enabled automatically by default. As of Rails 8.1, it is enabled for non-local environments (i.e., `!Rails.env.local?`).

```ruby
# Disable YJIT in memory-constrained environments
config.yjit = false

# Enable with options
config.yjit = { stats: true }
```

| Starting with version | Default |
|---|---|
| (original) | `false` |
| 7.2 | `true` |
| 8.1 | `!Rails.env.local?` |
</Accordion>

## Configuring Rails Components

Pass settings to individual Rails components through the same `config` object:

<Accordion title="Active Record">
```ruby
# Use Ruby format for schema dumps (database-independent)
config.active_record.schema_format = :ruby

# Raise error on page load if there are pending migrations (development)
config.active_record.migration_error = :page_load

# Log verbose query information
config.active_record.verbose_query_logs = true

# Append SQLcommenter tags to queries
config.active_record.query_log_tags_enabled = true

# Do not dump schema after migrations (production)
config.active_record.dump_schema_after_migration = false

# Limit inspect output to :id in production
config.active_record.attributes_for_inspect = [ :id ]
```
</Accordion>

<Accordion title="Action Mailer">
```ruby
# Development: don't raise errors if delivery fails
config.action_mailer.raise_delivery_errors = false

# Set host for URLs in mailer templates
config.action_mailer.default_url_options = { host: "example.com" }

# Production SMTP settings (use credentials for secrets)
config.action_mailer.smtp_settings = {
  user_name: Rails.app.credentials.dig(:smtp, :user_name),
  password:  Rails.app.credentials.dig(:smtp, :password),
  address:   "smtp.example.com",
  port:      587,
  authentication: :plain
}
```
</Accordion>

<Accordion title="Assets">
| Option | Description |
|---|---|
| `config.assets.precompile` | Additional assets to precompile beyond `application.css` and `application.js` |
| `config.assets.prefix` | Prefix where assets are served from; defaults to `/assets` |
| `config.assets.digest` | Enable SHA256 fingerprints in asset names; defaults to `true` |
| `config.assets.gzip` | Create gzipped versions of compiled assets; defaults to `true` |
| `config.assets.quiet` | Suppress logging of asset requests; defaults to `true` in development |

```ruby
# Precompile additional files
config.assets.precompile += %w[admin.js admin.css]
```
</Accordion>

## Database Configuration

Database configuration lives in `config/database.yml` and is loaded via `config/application.rb`. Active Record connections are managed by `ActiveRecord::ConnectionAdapters::ConnectionPool`.

```yaml
# config/database.yml
development:
  adapter: sqlite3
  database: storage/development.sqlite3
  pool: 5
  timeout: 5000

production:
  adapter: postgresql
  encoding: unicode
  database: myapp_production
  pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
  username: <%= ENV["MYAPP_DATABASE_USERNAME"] %>
  password: <%= ENV["MYAPP_DATABASE_PASSWORD"] %>
  host: <%= ENV["MYAPP_DATABASE_HOST"] %>
```

<Note>
The connection pool limit defaults to `5`. If you are running in a multi-threaded environment, set `pool` to at least the number of threads. If `ActiveRecord::ConnectionTimeoutError` is raised, increase the pool size in `database.yml`.
</Note>

You can also configure the database URL via an environment variable:

```bash
DATABASE_URL=postgres://user:password@host/myapp_production
```

## Custom Configuration

Use `config.x` for nested custom configuration, or `config` directly for single-level options:

```ruby
# config/application.rb
config.x.payment_processing.schedule = :daily
config.x.payment_processing.retries  = 3
config.super_debugger = true
```

Access it anywhere in the application:

```ruby
Rails.configuration.x.payment_processing.schedule # => :daily
Rails.configuration.x.payment_processing.retries  # => 3
Rails.configuration.x.payment_processing.not_set  # => nil
Rails.configuration.super_debugger                # => true
```

Load entire YAML config files with `config_for`:

```yaml
# config/payment.yml
production:
  environment: production
  merchant_id: production_merchant_id
  public_key:  production_public_key
  private_key: production_private_key

development:
  environment: sandbox
  merchant_id: development_merchant_id
  public_key:  development_public_key
  private_key: development_private_key
```

```ruby
# config/application.rb
module MyApp
  class Application < Rails::Application
    config.payment = config_for(:payment)
  end
end

# Anywhere in the app
Rails.configuration.payment["merchant_id"] # => environment-specific value
```

## Configuring Middleware

```ruby
# Add custom middleware to the end of the stack
config.middleware.use Magical::Unicorns

# Insert before a specific middleware
config.middleware.insert_before Rack::Head, Magical::Unicorns

# Insert after a specific middleware
config.middleware.insert_after Rack::Head, Magical::Unicorns

# Swap one middleware for another
config.middleware.swap ActionController::Failsafe, Lifo::Failsafe

# Remove a middleware entirely
config.middleware.delete Rack::MethodOverride
```

## Initialization Events

Rails provides 5 initialization hooks in the order they run:

| Hook | Description |
|---|---|
| `before_configuration` | Runs when the application class inherits from `Rails::Application` |
| `before_initialize` | Runs directly before the initialization process begins |
| `to_prepare` | Runs after all Railtie initializers; on every request in development, once in production |
| `before_eager_load` | Runs directly before eager loading |
| `after_initialize` | Runs after all application initializers complete |

```ruby
module YourApp
  class Application < Rails::Application
    config.before_initialize do
      # runs early in boot
    end

    config.after_initialize do
      # runs after everything is set up
    end
  end
end
```
